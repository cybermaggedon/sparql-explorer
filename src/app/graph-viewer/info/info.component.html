Welcome to SPARQL Sleuth!

This tool is used to explore a knowledge graph. A knowledge graph is a representation of information as a set of nodes, interconnected by edges. Nodes represent a single object. Nodes may have properties associated with them. Edges represent a relationsip between nodes, and have a direction - they are drawn as an arrow, and are labelled with an edge label which describes the relationship.

SPARQL Sleuth is an exploration tool. The ideas is that it helps you find out more information about the knowledge in the graph, starting with nodes representing knowledge you know, so that you can navigate to find out more about what is in the graph.

Getting started

When you start the application, you are faced with a blank canvas, so your first task is find information you know to start navigating.

There are three things you can explore straight away:

  • You can select Search from the menu, and search for information based on text in properties associated with a node.
  • You can select Schema and find out the names of class objects which are stored in the graph. Class objects represent the type of an object. For instance, the PostalAddress object represents the type of objects which are street addresses. Once you have found a class of interest, you can navigate to find objects which are associated with that type. Note that some type objects may not represent any information in the graph.
  • You can select Datasets and read more about datasets stored in the knowledge graph. Information about Datasets is represented as a node in the graph, so in some graphs you may be able to navigate from the Dataset node to find nodes stored in the dataset.

Search

The search tool is a simple way to add new nodes to your graph view. After selecting Search from the menu, you can enter a search term into the search dialog, and click Search to submit the search.

The dialog will then contain a table of information which was found. In the right-hand column is a column of textual properties which match your search term. In the left handle column is a list of nodes. In the middle column is the name of the edge relationship which connects the left-hand node to the property text.

Note that nodes may be listed multiple times, if there are multiple properties which match your search term.

When you have found some nodes of interest, clicking on the name in the left-hand column adds the node to the main graph display. Nodes are only added once to the graph. If you try to add a node which is already included in the main graph display, nothing will happen. Note that you may need to scroll the main display, or close the search dialogue to find the nodes.

Node interaction

Once you have some graph nodes to work with, you can Click on a graph node to see detail. Any detail for a node is shown in a property dialogue.

In this dialogue, you can click on a node and press re-centre, this clears the entire graph leaving only the selected node. This can be useful if your graph display is cluttered and you want to continue exploring with less information on the screen.

Nodes can have a URL associate with them, linking to a web page containing further information. Nodes may also have image information associate, in which case a preview thumbnail image will be shown. Images are not stored in the knowledge graph, SPARQL Sleuth shows images by following the image URL links.

Also in this dialogue are navigation buttons to allow you to explore nodes related to this node through relationship edges.

Node interaction

The relationship buttons are shown on the node display and have arrow icons ⮕. Clicking one of these relationship buttons causes SPARQL Sleuth to follow the edges associated with that relationship and add a set of related nodes and edges to the graph display.

Note when exploring relationships, all relationships have a direction (indicated by the direction of arrow-heads on the lines. The direction of the arrow on the button indicates which direction to explore.

  • When the arrow icon is on the left of the button, SPARQL Sleuth will look for relationships which are 'incoming' i.e. the arrow-head points to the selected node.
  • When the arrow icon is on the right of the button, SPARQL Sleuth will look for relationships which are 'outgoing' i.e. the arrow-head points away from the selected node.

By way of an example, if you select a node and then press the button marked type ⮕, you will explore type relationships with the selected node at the start of the arrow. This is essentially asking "What type is this node?".

If you select a node and select the button marked ⮕ type, you will explore type relationships with the selected node at the end of the arrow. This is essentially asking "Show me all nodes which have this type?".

Select the menu icon and click Schema to see a list of graph classes. Selecting the name of a class will cause that class to be added to the main graph view. Once you have that class added, you can use it to navigate to nodes which declare that class as their type. Note that there may be classes in the schema with no linked information, so you should look for classes which you think exist in the dataset.

The Datasets dialogue shows you information about the datasets in the graph. Note that not all knowledge graphs will have dataset information included. The Dataset schema used is the schema.org Dataset schema. Dataset information will be shown if it conforms to this schema.

The Datasets dialogue shows dataset descriptions, along with any source / author information and possible keywords. There is a small blue button which will add the Dataset object to the main graph view. It is possible that you will be able to navigate from the Dataset object to other information in the dataset.
